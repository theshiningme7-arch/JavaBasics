ðŸ”¹ METHODS IN JAVA

Definition:
A method is a block of code that performs a specific task.  
It helps in reusing code and organizing logic.

Purpose:
- Avoid repetition
- Improve readability
- Divide program into smaller parts

---

1. METHOD DECLARATION

Syntax:
returnType methodName(parameters) {
   // body
   return value;
}

Example:
int add(int a, int b) {
   return a + b;
}

---

2. METHOD CALLING

Flowchart:
Define method â†’ Call method â†’ Execute â†’ Return result

Example:
int sum = add(5, 7);
System.out.println(sum); // 12

---

3. TYPES OF METHODS

- Predefined methods (library methods)
   Example: Math.sqrt(16) â†’ 4

- User-defined methods
   Example: add(), multiply(), etc.

---

4. STATIC VS NON-STATIC METHODS

Static:
- Belong to class
- Called without creating object
Example: Math.max(10, 20)

Non-static:
- Belong to object
- Need object creation
Example:
class Student {
   void display() { System.out.println("Hello"); }
}
Student s = new Student();
s.display();

---

5. METHOD OVERLOADING

Definition:
Same method name, different parameters.

Example:
class Calculator {
   int add(int a, int b) { return a+b; }
   double add(double a, double b) { return a+b; }
}

---

Practice:
Q: Why use methods?
A: To reuse code and make programs modular.

ðŸ”¹ STRINGS IN JAVA

Definition:
A String is a sequence of characters enclosed in double quotes.  
Strings are objects of the `String` class.

Example:
String name = "Suprada";

---

1. STRING CREATION

Two ways:
- Using string literal: String s = "Hello";
- Using new keyword: String s = new String("Hello");

---

2. STRING METHODS

Common methods:
- length() â†’ returns length
- charAt(index) â†’ returns character
- substring(start, end) â†’ returns part of string
- equals(str) â†’ compares content
- toUpperCase() / toLowerCase()
- concat(str) â†’ joins strings

Example:
String s = "Java";
System.out.println(s.length());       // 4
System.out.println(s.charAt(2));      // v
System.out.println(s.concat(" Rocks")); // Java Rocks

---

3. STRING IMMUTABILITY

Rule:
Strings cannot be changed once created.  
Any modification creates a new object.

Example:
String s = "Hello";
s = s.concat(" World");
System.out.println(s); // Hello World

---

4. STRINGBUILDER & STRINGBUFFER

Definition:
Used for mutable (changeable) strings.

- StringBuilder â†’ faster, not thread-safe
- StringBuffer â†’ slower, thread-safe

Example:
StringBuilder sb = new StringBuilder("Hi");
sb.append(" Java");
System.out.println(sb); // Hi Java

---

5. STRING COMPARISON

equals() â†’ compares content  
== â†’ compares references (memory addresses)

Example:
String a = "Java";
String b = "Java";
System.out.println(a == b);     // true (same reference)
System.out.println(a.equals(b));// true (same content)

String c = new String("Java");
System.out.println(a == c);     // false (different objects)
System.out.println(a.equals(c));// true (same content)

---

Practice:
Q: Why are Strings immutable?
A: For security, efficiency, and caching.
