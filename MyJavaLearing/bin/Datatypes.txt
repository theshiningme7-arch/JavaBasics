#  Java Quick Notes

### Why Java Was Created
- Created in 1991 by James Gosling (Sun Microsystems).  
- Problems with C/C++: platform dependent, manual memory management, unsafe for internet, complex.  
- Java solved this with: secure, simple, reliable, platform independent language.  
- Motto: **WORA â€“ Write Once, Run Anywhere**.

---

### Platform Dependent vs Independent
- **Platform = OS + Hardware**  
- C/C++: Source â†’ Machine Code â†’ Runs only on that OS.  
- Java: Source â†’ Bytecode â†’ JVM â†’ Machine Code â†’ Runs anywhere.

---

### Java Execution Process
1. Write code â†’ `file.java`  
2. Compile â†’ `javac file.java` â†’ `file.class` (bytecode)  
3. Run â†’ `java file` â†’ JVM executes bytecode.  

---

### JVM, JRE, JDK
- **JVM**: Runs bytecode.  
- **JRE**: JVM + libraries (to run programs).  
- **JDK**: JRE + compiler + tools (to develop programs).  
- To **develop** â†’ JDK, to **run only** â†’ JRE.

---

### Bytecode
- Intermediate code after compilation.  
- Not machine code, understood by JVM.  
- Reason Java is platform independent.

---

### Compiled or Interpreted?
- Both.  
- Compiled â†’ `javac` â†’ bytecode.  
- Interpreted â†’ JVM executes bytecode.  
- JIT compiler â†’ converts frequent bytecode to native machine code for speed.

---

### Why Java is Secure
- No pointers.  
- Automatic memory management (Garbage Collector).  
- Bytecode verification.  
- Sandboxed execution.

###Objects
- An object is an instance of a class.
- It represents a real-world entity with state (data/fields) and behavior (methods).
- Example: If Car is a class, then myCar is an object of that class.
- Objects are created using the new keyword, e.g., Car myCar = new Car();

###Classes
- A class is a blueprint or template used to create objects.
- It defines the fields (variables) and methods (functions) that its objects will have.

class Car {
    String color;
    void drive() {
        System.out.println("Car is driving");
    }
}

 ###Methods
- A method is a block of code inside a class that performs a specific task.
- Methods define the behavior of objects.
- They can take parameters, return values, or simply execute logic.
void startEngine() {
    System.out.println("Engine started");
}
