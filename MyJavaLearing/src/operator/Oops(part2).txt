Definition:
Association defines a relationship between two classes.  
It shows how objects of one class are connected to objects of another class.

Purpose:
- Model real-world relationships
- Improve modularity
- Clarify how classes interact

Flowchart:
Class A â†” Class B â†’ Relationship established

Example:

   String name;
}

   String title;
}

Here, Student and Course can be associated (a student enrolls in a course).

ðŸ”¹ SPECIALIZED ASSOCIATION TYPES

1. AGGREGATION
Definition:
Represents a "HAS-A" relationship.  
Child objects can exist independently of parent.

Example:

   String name;
}

   List<Department> departments;
}

Flow:
University â†’ has â†’ Departments

---

2. COMPOSITION
Definition:
Represents a "HAS-A" relationship with strong dependency.  
Child objects cannot exist without parent.

Example:

   String type;
}
class Car {
   Engine engine = new Engine();
}

Flow:
Car â†’ has â†’ Engine  
If Car is destroyed, Engine is destroyed too.
